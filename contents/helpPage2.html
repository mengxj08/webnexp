Tutorial: #1 Define the research question:

Step 1.1: Start with a General Question:

How does XXX-Pod compare with iPod’s menu in terms of performance?

See more examples

Step 1.2: Define Target Population:

General question: How does earPod compare with iPod’s menu in terms of performance?

Target population? Question: we designed earPod for whom?

Step 1.3: Define Task(s) :

Task(s): menu selection. However, the menu selection task has endless possibilities:

  • single short menus
  • single long menus
  • hierarchical menus
Key insight: experiment design need to decide what subset of tasks is appropriate to test.

Step 1.4: Define Measures :

General question: How does earPod compare with iPod’s menu in terms of performance?

Measures: performance.

In HCI, we typically use three measures to quantify performance:

  • Speed
  • Accuracy
  • Learnability
Key insight: need to define "testable" measures.

Step 1.5: Define (other) Factors :

Factors: other than the different type of tasks, what other factors can influence the measures? Again: the number of factors are unlimited …

  • Scenario of use
  • Input device
  • Background of the user
  • Educational level
  • Gender
  • Ethnic background
  • Age
  • …
Key insight: experiment design need to determine a subset of factors to test.

Hypothesis Example: "XXX-Pod" menu vs. "iPod" menu

    Experiment Name: "XXX-Pod" menu vs. "iPod" menu
    Author: Tom
    Description: We design a new type of iPod named "XXX-Pod". This controlled experiment aims to prove that selecting menus using the "XXX-Pod" is better than using "iPod".

    Hypothesis: XXX-pod 
    is better than iPod
    in selecting menus
    under different sizes of screen
    based on speed and accuracy.

Experiment 2: "Opti" vs. "Qwerty" Keyboard

    Experiment Name: "Opti" vs. "Qwerty" Keyboard
    Author: Sam
    Description: We find a new layout of keyboard named "Opti". This controlled experiment aims to prove that typing using the "Opti" keyboard is faster than using the traditional "Qwerty"  Keyboard.

    1.1: General question
    how does the "opti" keyboard layout compare with the "qwerty" keyboard in performance? 
    1.2: Target Population?
    Computer users? 
    1.3: Task(s)?
    Type "the quick brown fox jumps over the lazy dog"
    1.4: Measure(s)?
    Speed, accuracy, learning 
    1.5: (Other) Factors? 
    Device: Touch typing vs. stylus?
    Screen size: different screen size?
